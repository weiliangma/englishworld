# EnglishWorld — 交互原型 & 用户旅程

> 你家孩子的一天：放学回家，掏出手机，打开 EnglishWorld

---

## 一、用户旅程地图

### 典型场景

```
放学回家 → 掏出手机 → 看到宠物 push 通知
→ "主人，我饿了..." 
→ 打开 App → 先喂宠物（金币不够了）
→ 去打两局题赚金币 → 回来再喂一次
→ 宠物开心了冒爱心 → 满足地关掉
→ 第二天继续
```

### 核心循环

```
登录 → 看宠物状态（发现饿了）→ 喂宠物（金币 -10）→
宠物笑了 → 开心 → 但金币少了 → 去打几题赚金币 →
答题赚金币 → 回来再喂/陪玩 → 宠物进化/换皮肤 →
更有动力 → 明天继续
```

## 三、新手引导（3 步）

```
Step 1: "给你的勇士起个名字"
        [输入框] 默认: "语法勇士"
        [下一步]

Step 2: "选一只陪你冒险的宠物"
        🐉 小火龙  🗡️ 小剑士
        🪄 小法杖  👻 小幽灵
        点击选择 → 起名字 → [开始冒险]

Step 3: "看到题目选答案就行，别慌"
        第一题没有倒计时
        答对后：🎉 "就这样，简单吧！"
        然后进入正常模式
```

## 四、宠物触发机制

| 触发时机 | 宠物表现 | 作用 |
|---|---|---|
| 每天首次打开 | 跳出来迎接，冒星星 | 暖开场 |
| 答对 5 题连击 | 撒花转圈 | 正向激励 |
| 答错 3 题 | 做加油姿势，"没关系！" | 减压 |
| Boss 战时 | 在角落挥舞小旗 | 陪伴感 |
| Boss 胜利 | 和角色抱在一起转 | 庆祝 |
| 长时间未登录 | 垂头丧气，"你终于来了" | 内疚驱动 |
| 宠物饿/脏了 | 头顶冒文字气泡 | 提醒照顾 |
| 购买新皮肤 | 立刻穿上转圈展示 | 即时满足 |
| 进化时 | 全屏进化动画 + 特效 | 里程碑仪式感 |

## 五、上线检查清单（单人版）

| # | 检查项 | 优先级 |
|---|---|---|
| 1 | 不用安装，打开即玩 | 必须 |
| 2 | 至少 2 个语法点（时态 + 冠词） | 必须 |
| 3 | 每题有反馈动画（正确/错误） | 必须 |
| 4 | 金币和经验系统 | 必须 |
| 5 | 宠物系统（喂食/陪玩/洗澡/状态显示） | 必须 |
| 6 | 手机触摸体验流畅 | 必须 |
| 7 | 能本地存进度（下次打开不丢） | 必须 |
| 8 | PWA 可添加到桌面 | 应有 |
| 9 | Boss 战（至少 1 个） | 应有 |
| 10 | 宠物商店（至少 5 件物品） | 应有 |
| 11 | 宠物进化（至少 2 个阶段） | 应有 |
| 12 | 战利品宝库（错题本） | 应有 |
