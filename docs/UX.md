# EnglishWorld — 交互原型 & 用户旅程

> 你家孩子的一天：放学回家，掏出手机，打开 EnglishWorld

---

## 一、用户旅程地图

### 典型场景

```
放学回家 → 掏出手机 → 看到宠物 push 通知
→ "主人，我饿了..." 
→ 打开 App → 先喂宠物（金币不够了）
→ 去打两局题赚金币 → 回来再喂一次
→ 宠物开心了冒爱心 → 满足地关掉
→ 第二天继续
```

### 核心循环

```
登录 → 看宠物状态（发现饿了）→ 喂宠物（金币 -10）→
宠物笑了 → 开心 → 但金币少了 → 去打几题赚金币 →
答题赚金币 → 回来再喂/陪玩 → 宠物进化/换皮肤 →
更有动力 → 明天继续
```

## 四、宠物触发机制

| 触发时机 | 宠物表现 | 作用 |
|---|---|---|
| 每天首次打开 | 跳出来迎接，冒星星 | 暖开场 |
| 答对 5 题连击 | 撒花转圈 | 正向激励 |
| 答错 3 题 | 做加油姿势，"没关系！" | 减压 |
| Boss 战时 | 在角落挥舞小旗 | 陪伴感 |
| Boss 胜利 | 和角色抱在一起转 | 庆祝 |
| 长时间未登录 | 垂头丧气，"你终于来了" | 内疚驱动 |
| 宠物饿/脏了 | 头顶冒文字气泡 | 提醒照顾 |
| 购买新皮肤 | 立刻穿上转圈展示 | 即时满足 |
| 进化时 | 全屏进化动画 + 特效 | 里程碑仪式感 |
| 收集 10 个新单词 | 宠物学会一个新词，对话框里蹦出来用 | 词汇激励 |
| 集齐一个主题所有单词 | 宠物获得该主题专属装饰（穿戴上） | 收集成就感 |
